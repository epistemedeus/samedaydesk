# SameDayDesk brand assets

The mark is the "Same-Day Stamp": a rounded-square dispatch tile with a knockout
double-chevron (fast-forward = same-day speed). One geometry, two color modes.

## Sources (edit these, then re-run the build step below)

- `icon-master.svg` : app-icon master (near-black tile + lime chevron). Universal, reads on any wallpaper.
- `icon-maskable.svg` : full-bleed master for Android adaptive icons (mark inside the center safe circle).
- `profile-dark.svg` : profile picture for dark backgrounds (lime chevron on near-black).
- `profile-light.svg` : profile picture for light backgrounds (oxide chevron + registration ring on bone).

The in-app mark is `client/src/components/BrandMark.tsx` (token-driven, themes itself).
The browser-tab favicon is `client/public/favicon.svg` (adapts to the browser's light/dark chrome).

## Profile pictures (ready to upload)

In `export/`:

- `profile-light-512.png`, `profile-light-400.png`

Pick by the surface you are posting on:

- Fiverr seller/agency avatar, LinkedIn Company page, Upwork AGENCY profile: either, depending on the surface's background.
- Upwork personal and LinkedIn personal profiles require a real face photo, so use a headshot there and keep the mark on cover images, gig thumbnails, and deliverables.

These are square; platforms crop to a circle. The mark sits inside the circular safe zone, so nothing important is cut.

## Social share card (og.png)

`client/public/og.png` (1200x630, rendered at 2x) is generated by `make-og.mjs`, which embeds
the real brand fonts into an SVG so it rasterizes accurately. To regenerate (from `client/`):

```bash
node brand/make-og.mjs
qlmanage -t -s 2400 -o /tmp/og brand/og.svg
sips -c 1260 2400 /tmp/og/og.svg.png --out public/og.png
rm brand/og.svg
```

The `og:image` tags in `client/index.html` already point to `/og.png`, so no markup change is needed.

## Regenerating the PNGs

The rasters in `client/public/` (`favicon.ico`, `apple-touch-icon.png`, `icon-192.png`,
`icon-512.png`, `icon-512-maskable.png`) and the `export/` profile PNGs were rendered from the
SVG sources above via macOS `qlmanage` (WebKit) plus `sips` for resizing. If you change a source
SVG, re-render with the same approach, or use any SVG rasterizer (sharp-cli, resvg, rsvg-convert).
