AI Search Readiness Kit

Everything you need to make a website readable — and quotable — by AI search engines (ChatGPT, Perplexity, Claude, Google AI Overviews). The benchmark, the exact templates, and a prioritized checklist. Apply it in an afternoon.

1 · Why this matters (the 189-company benchmark)

AI search can only quote pages it can actually crawl and parse. We scored the homepages of 189 well-known companies across 10 industries on six fundamentals: AI-crawler access, JSON-LD structured data, title/meta, Open Graph, an XML sitemap, and llms.txt. Average score by industry:

Names that surprised us: OpenAI and GitHub scored a D (JS-heavy homepages, thin server-rendered content); Perplexity — an AI search engine — a C; LlamaIndex, whose whole product is making data readable by LLMs, a D; Klarna an F (38, the lowest). The gaps are almost always the same three: JavaScript-only rendering, no JSON-LD, no sitemap. All fixable below. Raw data (CC-BY, 189 rows): CSV.

2 · The templates (copy, fill the REPLACE blanks, paste)

2a · Organization JSON-LD — into your homepage <head>

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "REPLACE_COMPANY_NAME",
  "url": "https://REPLACE_DOMAIN/",
  "logo": "https://REPLACE_DOMAIN/logo.png",
  "description": "REPLACE_ONE_SENTENCE_WHAT_YOU_DO",
  "sameAs": [
    "https://www.linkedin.com/company/REPLACE",
    "https://x.com/REPLACE"
  ]
}
</script>

2b · FAQPage JSON-LD — engines quote these directly

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    { "@type": "Question", "name": "REPLACE_What does REPLACE_COMPANY do?",
      "acceptedAnswer": { "@type": "Answer", "text": "REPLACE_Clear 1-2 sentence answer." } },
    { "@type": "Question", "name": "REPLACE_How much does it cost?",
      "acceptedAnswer": { "@type": "Answer", "text": "REPLACE_Plain pricing answer." } },
    { "@type": "Question", "name": "REPLACE_Who is it for?",
      "acceptedAnswer": { "@type": "Answer", "text": "REPLACE_Ideal-customer answer." } }
  ]
}
</script>

2d · /sitemap.xml — so pages get discovered

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url><loc>https://REPLACE_DOMAIN/</loc></url>
  <url><loc>https://REPLACE_DOMAIN/pricing</loc></url>
  <url><loc>https://REPLACE_DOMAIN/about</loc></url>
</urlset>

2e · Title, meta & Open Graph — into <head>

<title>REPLACE_Company — REPLACE_what you do in 5 words</title>
<meta name="description" content="REPLACE_One clear sentence a person (or an AI) could quote." />
<meta property="og:title" content="REPLACE_Company — REPLACE_what you do" />
<meta property="og:description" content="REPLACE_Same clear sentence." />
<meta property="og:type" content="website" />
<meta property="og:url" content="https://REPLACE_DOMAIN/" />
<meta name="twitter:card" content="summary_large_image" />

3 · The prioritized checklist

1. Server-render your core copy. If your homepage is a JS shell, AI crawlers see nothing. Ensure the headline, what-you-do sentence, and key facts are in the initial HTML (SSR/SSG or prerender).
2. Add Organization + FAQPage JSON-LD (2a, 2b). This is the single highest-leverage fix — it gives engines machine-readable facts to quote.
3. Replace /robots.txt (2c) so GPTBot / PerplexityBot / ClaudeBot / Google-Extended are explicitly allowed.
4. Publish /sitemap.xml (2d) and submit it in Bing Webmaster Tools (ChatGPT reads Bing's index).
5. Fix title + meta + Open Graph (2e) — one clear, quotable sentence beats clever copy.
6. Add a short /llms.txt with your name, what you do, pricing, and links — a clean summary for LLMs.
7. Re-check. Run your site through the free checker and confirm each gap is now green.
